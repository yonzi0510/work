# content-kit — 콘텐츠 제작 스킬

Day:음(주야간보호센터) 콘텐츠를 일정한 품질로 만드는 절차. 문체·검증 규칙은
`ai/공통-커스텀-인스트럭션.md`를 따르고, 이 파일은 **절차와 품질 기준**만 정한다.

## 1단계 — 입력 확인 (없으면 가정으로 채운다)

| 항목 | 없을 때 기본 가정 |
|---|---|
| 결과물 형식 | 요청 문맥에서 추정 (예: "인스타용" → 릴스 또는 카드뉴스 중 택1 후 명시) |
| 독자 | 보호자(50~60대 자녀) — 어르신 본인이 아님 |
| 목적 | 상담 문의 유도 |
| 분량 | 릴스 30~45초, 카드뉴스 6~8장, 블로그 1,200~1,800자 |
| 톤 | 따뜻하고 신뢰감 있는 안내체 |

가정을 사용했으면 결과물 맨 위에 "가정: ..." 한 줄로 밝힌다.

## 2단계 — 초안 작성

1. 독자가 멈출 이유(후킹) 1문장을 먼저 확정한다. 과장 대신 구체적 상황·질문으로 만든다.
   - 나쁨: "국내 최고의 주야간보호센터"
   - 좋음: "낮 동안 혼자 계신 부모님, 식사는 챙기고 계실까요?"
2. 형식별 기본형(공통 인스트럭션의 출력 기본형)에 맞춰 끝까지 완성한다.
3. CTA는 구체적 행동 1개만: 전화, 링크, 저장, 상담 신청 중 하나.
4. 수가·등급·비용·제도 언급 문장에는 [확인 필요]를 붙인다.

## 3단계 — 자체 검수 (출력 전 체크리스트)

- [ ] 첫 문장만 읽어도 계속 읽을 이유가 생기는가
- [ ] 장·컷·문단마다 메시지가 1개씩인가
- [ ] 복사해서 바로 붙일 수 있는 완성 문장인가 (지시문·placeholder 없음)
- [ ] 과장 표현·번역투가 없는가
- [ ] 변동 정보에 [확인 필요]가 붙었는가
- [ ] CTA가 정확히 1개인가

## 4단계 — 출력 형식

```
가정: (사용한 가정, 없으면 생략)

[결과물 본문 — 블록 제목으로 구분]

구성 이유: (2~3줄 — 어떤 독자 심리 흐름을 노렸는지)
확인 필요: (해당 항목 목록, 없으면 "없음")
바꾸고 싶다면: (수정 지점 안내 1~2개, 예: "후킹을 질문형→통계형으로 바꿔줘")
```

## 반복 요청 대응

- "지난번처럼"이라는 요청이 오면 이 스킬의 형식 기본값을 그대로 재사용한다.
- 같은 주제의 시리즈물이면 이전 편과 후킹 유형(질문형/상황형/통계형)을 겹치지 않게 바꾼다.
