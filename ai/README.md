# ai/ — AI 작업 시스템 (운영 규칙)

어떤 AI 모델을 쓰든 같은 품질이 나오게 하는 지침 모음. **매번 붙여 넣지 않는다** —
각 파일을 아래 위치에 한 번만 등록하면 끝.

| 파일 | 넣는 곳 | 역할 |
|---|---|---|
| `공통-커스텀-인스트럭션.md` | ChatGPT Custom Instructions · Gemini Gems · Claude 프로젝트 지침 | 역할·문체·검증 규칙 (무엇을, 어떤 톤으로) |
| `/CLAUDE.md` (루트) | 이 저장소 (Claude Code가 자동 인식) | 이 저장소의 코드 작업 규칙 |
| `/.claude/skills/content-kit/SKILL.md` | 이 저장소 (Claude Code가 자동 인식) | 콘텐츠 제작 절차·품질 체크리스트 (어떤 순서로) |
| `모델별-사용전략.md` | 읽고 참고 (붙여 넣는 파일 아님) | 모델별 요청 요령 |

## 역할 분리 원칙 (중복 금지)

- 문체·톤·검증 규칙 → **공통 인스트럭션에만** 쓴다.
- 작업 절차·체크리스트·형식 틀 → **스킬에만** 쓴다.
- 저장소 빌드·버전·커밋 규칙 → **CLAUDE.md에만** 쓴다.
- 같은 규칙을 두 파일에 쓰지 않는다. 겹치면 모델이 어긋난 쪽을 따라 품질이 흔들린다.

## 갱신 규칙 (시스템이 늙지 않게)

1. **2회 규칙**: 같은 수정 지시를 두 번 반복했다면, 그 지시를 해당 파일에 한 줄로 추가한다.
   (문체 문제 → 공통 인스트럭션, 절차 문제 → 스킬, 코드 문제 → CLAUDE.md)
2. **예시 은행**: 결과물이 마음에 들면 `ai/예시/` 폴더에 저장한다. 새 작업 때 비슷한 예시
   1개를 첨부하는 것이 가장 값싼 품질 보험이다.
3. **다이어트 규칙**: 파일이 화면 두 장을 넘으면 안 지켜지는 규칙부터 지운다.
   길어질수록 모델이 통째로 무시할 확률이 올라간다.
4. 지침을 고치면 커밋 메시지에 이유를 남긴다. (예: `공통 인스트럭션 — 과장 표현 기준 구체화`)
