# CLAUDE.md — work (Day:음 업무 도구 모음)

주야간보호센터 Day:음(dayeum.co.kr)에서 쓰는 웹 도구 저장소. GitHub Pages로 배포된다
(https://yonzi0510.github.io/work/). 사용자는 센터 직원(비개발자)이므로 UI 문구와 README는
쉬운 한국어 안내체로 쓴다.

## 도구

- `pg/` 프로그램 일지 생성기 — 운영기록지 "반응 및 특이사항" 문장 생성. 문장은행 내장 단일 HTML.
- `pg2/` 강사배치 달력 — 월별 강사배치표 작성·인쇄(A4 가로). 단일 HTML, localStorage 저장.

## 절대 규칙

- 각 도구는 **인터넷 없이 HTML 파일 하나만 열어도 동작**해야 한다. 외부 스크립트·API·CDN 의존을
  새로 추가하지 않는다(구글 폰트 링크는 예외 — 없어도 동작에 지장 없게 유지).
- pg 수정 경로: 문장 데이터는 `pg/문장은행.json`, 로직·UI는 `pg/template.html`에서 고친 뒤
  `build.py`로 빌드해 산출물을 `index.html`에 반영한다. **index.html을 직접 편집하지 않는다.**
- 일지 문장은 `~하심 / ~하셨음` 공문서체를 유지한다. 같은 회차 안에서 같은 문장이 중복
  배분되면 안 된다.
- 레이아웃 변경 시 인쇄(A4)와 모바일 화면을 함께 확인한다.
- 이용 어르신 개인정보(실명, 등급 등)를 코드·데이터·커밋에 넣지 않는다.

## 버전·커밋 규칙

- 버전 표기: `v{YYYY.MM.DD.NNN}` (그날의 작업 순번). HTML `<title>` 아래 주석과 헤더의
  `.ver` 표기를 **함께** 갱신한다.
- 커밋 메시지: `도구명 v버전 — 변경 요지 한 줄` (한국어).
  예: `프로그램 일지 생성기 v2026.07.06.007 — 총평 기반 생성으로 전환`
- 작업 완료 보고: 무엇이 왜 바뀌었는지 2~3줄 + 직원에게 안내할 한 줄 문구를 함께 제안한다.

## 새 도구를 추가할 때

- 폴더 하나 = 도구 하나. 직원용 사용법을 담은 `README.md`를 폴더에 만들고, 루트 README 표에
  링크를 추가한다.
- 기존 디자인 토큰을 재사용한다: `--navy #1F3864`, `--teal #1D9E75`, Noto Sans KR,
  카드형 레이아웃.
